字符串哈希_学习笔记

到现在才学习字符串哈希感觉真的有点晚了，还是填个坑

所谓字符串hash是一个很神奇的东西

对于字符串的题目，hash是一个很好的方法（不过是非正解）
有一句话是这么说的

字符串的题

简单的直接hash A掉

难的你也不会做（后缀数组，后缀自动机，ac自动机…学了也不会）也只有hash混分

字符串hash的时间复杂度比较小，功能也很强大，所以字符串hash这种神奇的东西一定要学啊

所谓字符串HASH就是把一个字符串变成一个数字
如何变成一个数字呢？
例如: 设p=255
给出 ‘abba’ 这个字符串
可以这么转化

那么abba这个字符串就可以化为一个数
至于怎么算

可以以O(n)的时间算出

这样就可以算出一个字符串 怎样用数字表示

此时因为这样算出来的数可能会很大
用int 和 long long 自然会溢出

但是溢出一样不要紧，如果两个穿相同，即使溢出的值也是相同的

用unsigened long long（ULL）比较好 因为是没有负数的原因可以到2的64次方 这样自然溢出

相当于对2的64次方取膜，冲突自然会小

注意

  1. hash取模一般模一个大质数:

    10的9次方+7或10的九次方+9，因为用质数的冲突概率比较小（我不会证），一般不会被卡掉

  2. 如果不行就双HASH:

    就是分别对10的9次方+7 &&10的九次方+9这两个数都取模

    如果两个串的HASH1值和HASH2值都相等，那么可以判断这两个串相等
    这样冲突的概率会很小很小（我不会算）

用字符串哈希的好处（本人新学不怎么知道）
1.这样就可以快速的比较出字符串是否相同
2.可以快速判断同一个字符串中的两段字符串是否相等
3.回文串判断（正的HASH一遍，反着HASH一遍，判断是否相等）
……..

LAST.最重要的一点：可以用字符串HASH混字符串的题的很多分！！
