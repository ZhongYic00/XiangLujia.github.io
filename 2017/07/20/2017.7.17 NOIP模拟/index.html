求和

题目描述

求 1^b+2^b+…+a^b 的和除以 10000 的余数

输入格式

第一行包含一个正整数 N 表示共有 N 组测试数据；
接下来 N 行，每行包含两个整数 a 和 b。

输出格式

共 N 行，每行一个对应的答案。

数据规模

对于 30%的数据，满足 N<=10，a，b<=1000;
对于 100%的数据，满足 N<=100，a，b<=1,000,000,000。

输入样例

1

2 3

输出样例

9

题解

因为a和b的最大范围都到了10^9，而且还有多组的数据
所以显然O(n*a)的复杂度是过不了的
这个时候就应该找一下此题的更优做法
因为mod值为10000，所以不难想到

(a^b)mod10000=((a mod10000)^b)mod10000

但是这样也是O(na)的复杂度
还是要优化，我反正打了O(na)的暴力

那么有这样的方法：
eg：a=310030
算出后面末尾的
310001^b+310002^b+310003^b…310031^b = 1^b+2^b+3^b…+31^b
然后前面的就是31个
1^b+2^b+… 999^b

这样就行了
ans 和sum 详细看注释

序列合并

题目描述

有两个长度都为 N 的序列 A 和 B，在 A 和 B 中各取一个数相加可以得到 N^2 个和，求这 N^2 个 和中最小的 N 个。

输入格式

第一行一个正整数 N；

第二行 N 个整数 Ai，满足 Ai<=Ai+1 且 Ai<=10^9；

第三行 N 个整数 Bi，满足 Bi<=Bi+1 且 Bi<=10^9；

输出格式

仅一行，包含 N 个整数，从小到大输出这 N 个最小的和，相邻数字之间用空格隔开。

数据规模

对于 50%的数据，满足 1<=N<=1000;

对于 100%的数据，满足 1<=N<=100000。

输入样例

3

2 6 6

1 4 8

输出样例

3 6 7

题解

注意到这个a队列和b队列都是非严格递增的
所以可以得到

A_1+B_1<=A_1+B_2<=A_1+B_3….<=A_1+B_n

A_2+B_1<=A_2+B_2<=A_2+B_3….<=A_2+B_n

A_3+B_1<=A_3+B_2<=A_3+B_3….<=A_3+B_n

A_i+B_1<=A_i+B_2<=A_i+B_3….<=A_i+B_n

有N个这样的有序队列

为比较出最小值，维护优先队列

每次把每个队列的第一个元素放入优先队列

然后取出最小值，如果这个最小值来自第k个队列中，那么把第k个队列的下一个元素放入队列中 这样队列中的会是最小的

Tower

题目描述

平面上有 N 个整数坐标点。如果将点（x0,y0）移动到(x1,y1)，则需要的代价为|x0-x1|+|y0-y1|。 求使得 K（K=1,2,3….N）个点在同一位置上最小需要的代价。

输入格式

第一行包含一个正整数 N；
接下来 N 行，每行两个正整数 xi 和 yi，为第 i 个点的坐标，不超过 10^6。

输出格式

共 N 行，第 i 行为使得有 i 个点在同一位置的最小代价。

数据规模

对于 100%的数据，满足 1<=N<=50。

输入样例

4
15 14
15 16
14 15
16 15

输出样例

0
2
3
4

题解

最优的解的位置一定是在(x_i, y_j)
总有一个点的行和最优点相等，总有一个点列和最优点相等
因为n<=50
所以直接枚举
复杂度O(n^4)可过
